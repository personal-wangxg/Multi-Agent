# SCENE-004：练习闭环场景

**文档版本**：v1.0  
**创建日期**：2026-06-15  
**场景编号**：SCENE-004  
**关联功能需求**：FR-07（节点内错题闭环）、FR-13（人机协同）

---

## 1. 场景概述

**一句话描述**：学生在虚拟教室或作业批改中答错题目时，系统自动启动"诊断→补充讲解→同类题→再评估"的四步微观闭环，直至学生达到掌握标准或达到最大重试次数。

**参与角色**：
- 错题诊断Agent
- 补充讲解Agent
- 同类题生成Agent
- 评估Agent
- 节点掌握状态更新Agent

### 2.1 四步详解

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         Step A: 错题诊断                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  输入：original_question + student_answer + correct_answer                 │
│                                                                             │
│  ┌───────────────────────────────────────────────────────────────────────┐  │
│  │ 诊断维度：                                                             │  │
│  │                                                                       │  │
│  │ error_type:                                                          │  │
│  │   - concept: 概念理解错误（如混淆了判别式条件）                         │  │
│  │   - computational: 计算错误（如符号处理失误）                          │  │
│  │   - reading: 审题错误（如看错方程个数）                                │  │
│  │   - prerequisite: 前置知识缺失                                        │  │
│  │                                                                       │  │
│  │ specific_cause: 自然语言描述具体原因                                  │  │
│  │ severity: low / medium / high                                        │  │
│  │ prerequisite_knowledge_gap: 若为prerequisite类型，指出缺失节点ID      │  │
│  └───────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│  输出：error_diagnosis_report                                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                         Step B: 补充讲解                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  根据error_type选择讲解策略：                                               │
│                                                                             │
│  ┌───────────────────────────────────────────────────────────────────────┐  │
│  │ case: concept                                                        │  │
│  │   → 返回该节点对应concept层核心讲解                                    │  │
│  │   → 使用类比/具象化方式重新解释                                      │  │
│  └───────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│  ┌───────────────────────────────────────────────────────────────────────┐  │
│  │ case: computational                                                  │  │
│  │   → 逐行计算步骤详解                                                  │  │
│  │   → 标注易错点提醒                                                    │  │
│  └───────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│  ┌───────────────────────────────────────────────────────────────────────┐  │
│  │ case: reading                                                        │  │
│  │   → 审题方法训练（圈关键词/翻译题意/拆分子问题）                      │  │
│  └───────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│  ┌───────────────────────────────────────────────────────────────────────┐  │
│  │ case: prerequisite                                                   │  │
│  │   → 推荐返回前置节点复习                                               │  │
│  │   → 此时不进入后续步骤，直接跳转建议                                  │  │
│  └───────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│  输出：supplementary_lesson（含文本+资源推荐）                              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                         Step C: 同类题生成                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  同类题生成策略：                                                           │
│  ┌───────────────────────────────────────────────────────────────────────┐  │
│  │ - 保持原题考点不变（如"消元法"）                                      │  │
│  │ - 变换数值/情境/表述方式                                              │  │
│  │ - 避免学生记忆答案而非理解方法                                        │  │
│  │ - 难度略低于原题（给信心）                                            │  │
│  │ - 每次生成1道，可循环生成                                            │  │
│  └───────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│  约束：                                                                    │
│  - 同类题不得复用原题数值                                                  │
│  - 连续3次生成失败则降级从题库随机抽取                                    │
│                                                                             │
│  输出：new_question（含题干+参考答案）                                    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                         Step D: 再评估                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  学生提交新题答案后，评估Agent：                                           │
│                                                                             │
│  ┌───────────────────────────────────────────────────────────────────────┐  │
│  │ 1. 对比正确答案，判定正确/错误                                        │  │
│  │ 2. 检查掌握条件：                                                    │  │
│  │    - 连续2次答对 → mastered                                          │  │
│  │    - 或累计正确率≥80% → mastered                                    │  │
│  │ 3. 未掌握但attempt_count < MAX：                                    │  │
│  │    → 回到Step A（诊断新错误原因）                                    │  │
│  │ 4. 未掌握且attempt_count ≥ MAX：                                    │  │
│  │    → 标记needs_teacher_attention                                    │  │
│  └───────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│  输出：loop_attempt_record                                                │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 3. Agent角色定义

| Agent ID | 角色名称 | 框架 | 职责说明 | 特殊要求 |
|----------|---------|------|---------|---------|
| `agent_scene004_diagnosis_001` | 错题诊断Agent | DeepAgents | 分析错误类型和原因 | 绑定`tpl_scene004_diagnosis_v1.0` |
| `agent_scene004_lesson_002` | 补充讲解Agent | DeepAgents | 根据错误类型生成针对性讲解 | 绑定`tpl_scene004_lesson_v1.0` |
| `agent_scene004_question_003` | 同类题生成Agent | DeepAgents | 生成保持考点不变的变式题 | 绑定`tpl_scene004_question_v1.0` |
| `agent_scene004_evaluation_004` | 评估Agent | DeepAgents | 判定掌握状态，决定循环走向 | 绑定`tpl_scene004_evaluation_v1.0` |

**协作模式**：Sequential Loop（四步顺序执行，Step D决定是否继续循环）

### 4.3 补充讲解输出

```yaml
supplementary_lesson:
  loop_id: "err_loop_20260615_001"
  node_id: "tool_elimination_method_06"
  
  lesson_content:
    title: "代入消元法计算规范讲解"
    
    sections:
      - type: "error_identification"
        duration_minutes: 2
        content: |
          让我们先找出错误在哪里。
          原方程组：2x + 3y = 13 ...(1)
                    3x + 2y = 12 ...(2)
          
          从(1)得：x = (13 - 3y) / 2  ← 注意：13 - 3y，不是 13 + 3y！
          
          学生错误地将 -3y 写成了 +3y
      
      - type: "correct_procedure"
        duration_minutes: 5
        content: |
          正确步骤：
          (1) 2x + 3y = 13
              x = (13 - 3y) / 2  ← 移项要变号！
          
          (2) 代入(1)：
              3(13 - 3y)/2 + 2y = 12
              39 - 9y + 4y = 24
              -5y = -15
              y = 3
          
          代回求 x：
              x = (13 - 3*3) / 2 = 4/2 = 2
      
      - type: "common_mistakes"
        duration_minutes: 3
        content: |
          易错点提醒：
          1. 移项必须变号：+3y → -3y，-2y → +2y
          2. 分数计算：注意分子分母同时处理
          3. 代入检查：用原方程验证解是否正确
  
  recommended_resources:
    - type: "worksheet"
      id: "res_calc_001"
      description: "消元法计算训练20题"
```

## 6. 关键约束

| 约束类型 | 约束内容 | 来源 |
|---------|---------|------|
| **闭环约束** | 答错题必须触发本场景错题闭环 | DP-ARCH-12 |
| **循环约束** | 默认最大重试次数=3 | FR-07 |
| **掌握标准** | 连续2次答对 或 累计正确率≥80% | FR-07 |
| **prerequisite处理** | 诊断为prerequisite类型时直接跳转前置节点 | FR-07 |
| **题库降级** | 连续3次生成失败则降级从题库抽取 | FR-07 |
| **状态更新** | 闭环完成后必须更新节点掌握状态 | FR-07 |
