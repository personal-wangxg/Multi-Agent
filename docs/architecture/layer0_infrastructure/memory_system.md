## 1. 设计目标

构建三层记忆体系，支持：
- 会话级短期记忆（STM）
- 工作级中期记忆（WM）
- 长期记忆（LTM）
- 记忆的写入、检索与管理

| 记忆类型 | 存储位置 | 生命周期 | 用途 |
|---------|---------|---------|------|
| **STM** | 内存 | 单会话 | 当前对话上下文 |
| **WM** | VFS/磁盘 | 当前任务 | 中间产物、决策索引 |
| **LTM** | 向量数据库 | 跨任务 | 模板、偏好、历史经验 |

```
│  │  · 历史教学经验                                        │   │
│  │  · 教师反馈记录                                        │   │
│  └─────────────────────────┬───────────────────────────────┘   │
│                            │                                  │
│                            ▼                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  工作记忆（WM）- VFS/磁盘                              │   │
│  │  · 当前任务的中间产物                                   │   │
│  │  · 决策索引与 Wiki 页面                                 │   │
│  │  · 完整阶段产物文件                                     │   │
│  └─────────────────────────┬───────────────────────────────┘   │
│                            │                                  │
│                            ▼                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  短期记忆（STM）- 内存                                  │   │
│  │  · 当前会话的对话历史                                   │   │
```

### 2.2 记忆管理器组件

| 组件 | 职责 | 核心功能 |
|-----|------|---------|
| **STM 管理器** | 管理会话级记忆 | 上下文压缩、Token 控制 |
| **WM 管理器** | 管理任务级记忆 | 文件读写、版本管理 |
| **LTM 管理器** | 管理长期记忆 | 向量检索、记忆写入 |
| **记忆协调器** | 协调三层记忆 | 记忆注入、策略选择 |

## 3. 短期记忆（STM）设计

### 3.1 STM 数据结构

```yaml
short_term_memory:
  session_id: "sess_001"
  conversation_id: "conv_001"
  
  # 对话历史
  messages:
    - message_id: "msg_001"
      role: "user"
      content: "帮我规划一门初三数学课程"
      timestamp: "2026-06-12T10:00:00Z"
    - message_id: "msg_002"
      role: "assistant"
      content: "好的，我来帮您规划..."
      timestamp: "2026-06-12T10:01:00Z"
  
  # 当前上下文状态
  context:
    current_task: "course_planning_stage1"
    decision_index_summary: "已确认决策：总课时12..."
    retrieved_memories: []
  
  # Token 统计
  token_count: 1500
  max_tokens: 8192
  
  # 元数据
  created_at: "2026-06-12T10:00:00Z"
  last_updated: "2026-06-12T10:05:00Z"
```

### 3.2 上下文压缩策略

| 策略 | 触发条件 | 操作 |
|-----|---------|------|
| **策略A：摘要压缩** | Token > 窗口 50% | 对早期消息生成摘要 |
| **策略B：旧消息遗忘** | Token > 窗口 70% | 移除最早的非关键消息 |
| **策略C：中间产物卸载** | Token > 窗口 80% | 将中间产物写入 WM |

## 4. 工作记忆（WM）设计

```

├── sessions/
│   └── sess_001/
│       ├── working/
│       │   ├── outline_draft.md
│       │   ├── kn_stage1.yaml

│       │   └── index.md
```

### 4.2 WM 核心接口

| 接口 | 功能 | 参数 | 返回值 |
|-----|------|------|--------|
| `write_file(path, content)` | 写入文件 | path: str, content: str | 写入结果 |
| `read_file(path)` | 读取文件 | path: str | 文件内容 |
| `list_files(directory)` | 列出文件 | directory: str | 文件列表 |
| `delete_file(path)` | 删除文件 | path: str | 删除结果 |
| `create_directory(path)` | 创建目录 | path: str | 创建结果 |

## 5. 长期记忆（LTM）设计

### 5.1 LTM 数据结构

```yaml
long_term_memory:
  memory_id: "mem_course_plan_template_math_001"
  type: "template"                    # template / preference / feedback / experience
  
  # 记忆内容
  content: |
    课程规划模板：
    - 总课时：12
    - 教学方法：讲授法 + 探究法
    - 评估方式：课堂小测 + 书面作业
  
  # 向量嵌入
  embedding: [0.1, 0.2, 0.3, ...]     # 向量表示
  
  # 元数据
  domain: "mathematics"
  grade_level: "9"
  created_at: "2026-06-01T10:00:00Z"
  updated_at: "2026-06-12T15:30:00Z"
  usage_count: 15
  rating: 0.85
```

### 5.2 LTM 检索流程

```
检索请求
       │
       ▼
[生成查询向量]
       │
       ▼
[向量数据库检索]
       │
       ▼
[返回 top_k 结果]
       │
       ▼
[过滤与排序]
       │
       ├─ 按相关性排序
       ├─ 过滤低评分记忆
       └─ 去重
       │
       ▼
[注入上下文]
       │
       └─ 每次注入 ≤ N 条（默认 3）
```

### 5.3 LTM 写入时机

| 时机 | 触发条件 | 写入内容 |
|-----|---------|---------|
| **任务完成时** | 教师确认最终产物 | 决策索引摘要、关键参数 |
| **反馈接收时** | 教师提交反馈 | 反馈内容、评分 |
| **模板沉淀时** | Agent 生成优质模板 | 完整模板内容 |

## 6. 记忆联动机制

### 6.1 记忆注入流程

```
Agent 启动
       │
       ▼
[读取决策索引摘要]
       │
       ▼
[注入 STM]
       │
       ▼
[从 LTM 检索相关记忆]
       │
       ▼
[注入 ≤ 3 条相关记忆]
       │
       ▼
[检查 Token 量]
       │
       └─ 超限 → 触发压缩策略
```

### 6.2 决策索引与记忆体系的关系

```
决策索引（WM）
       │
       ├─ 写入 LTM（跨任务复用）
       │
       └─ 摘要注入 STM（当前会话）
```

## 7. 技术选型

| 组件 | 推荐技术 | 理由 |
|-----|---------|------|
| STM | Python 内存对象 | 快速访问，会话结束后销毁 |
| WM | 文件系统 + Git | 持久化、版本管理 |
| LTM | Milvus / Pinecone | 向量检索、语义匹配 |
| 向量生成 | OpenAI Embeddings | 高质量嵌入 |

---

## 8. 关键设计决策

| 决策点 | 设计选择 | 理由 |
|-------|---------|------|
| 三层架构 | STM/WM/LTM 分离 | 各司其职，优化性能 |
| LTM 注入限制 | 每次 ≤ 3 条 | 控制上下文膨胀 |
| 决策索引摘要 | ≤ 300 Token | 最小化上下文占用 |
| WM 版本管理 | Git | 支持回溯、协作 |
