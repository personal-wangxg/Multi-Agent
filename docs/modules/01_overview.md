## 1. 文档概述

### 1.1 文档目的

本文档定义 **EduAgents 多智能体辅助教学框架** 的顶层系统需求，面向项目干系人、架构师、开发人员与测试人员，用于：

- 明确系统建设的目标与边界
- 指导后续功能需求、设计与实现
- 作为测试验收与版本迭代的基准

### 1.2 适用读者

**首要阅读对象：EduAgents 体系内的智能体（Agent）。** 本系统需求文档的结构、术语、示例优先服务于智能体的理解与执行。人类（教师、开发人员、项目干系人）为次要阅读对象，文档对人类保持可读性但不主导。

#### （4）你必须遵守的硬约束优先级

优先级从高到低，上一级未满足时直接拒绝执行下一级：

1. **工具权限**（4.10.4 tool_whitelist）：白名单外工具直接拒绝
2. **输出 Schema**（4.10.2 JSON Schema）：校验失败自动重试 ≤ 3 次
3. **上下文窗口**（5.6.3 Token 公式）：超过模型窗口 × 50% 必须先压缩
4. **任务边界**（4.10.1 template.task_scope.must_not_do）：禁止任务直接拒绝
5. **记忆预算**（5.6.5 retrieval_config.top_k）：记忆检索不超限
6. **Token 预算**（5.1 会话级 Token 上限）：接近阈值告警，达上限暂停

### 1.3 教育学理论基础

立体分层知识网络与动态学习路径的设计根植于以下教育学与认知科学理论基础。每一理论在系统中对应具体的设计取舍，且作为约束贯穿全部需求——任何功能需求如与下述理论主张冲突，应以理论主张为准调整设计。

| 理论来源 | 核心主张 | 在本系统中的体现 |
|---------|---------|----------------|
| **布鲁姆教育目标分类学（Bloom's Taxonomy）** | 认知领域目标从低到高依次为：记忆 → 理解 → 应用 → 分析 → 评价 → 创造。教育应覆盖完整层级而非停留在记忆层 | 概念层节点对应"分析/评价/创造"等高级认知目标；技能层对应"应用/分析"；工具层对应"记忆/理解/应用"。每个节点的 bloom_level 字段即源于此分类 |
| **社会建构主义（Social Constructivism, Vygotsky）** | 学习是学习者通过社会互动共同建构意义的过程；教师的角色是"脚手架"（scaffolding），在学习者的"最近发展区"（Zone of Proximal Development）提供支持并逐步撤出 | 虚拟教室（SCENE-003）中 Agent 扮演的是"会逐步降低支持的脚手架"而非知识灌输者；动态学习路径根据学生当前节点表现调整"下一步"的难度与支持度 |
| **认知负荷理论（Cognitive Load Theory, Sweller）** | 人类工作记忆容量有限（约 4±2 个信息组块），过高的内在认知负荷与外在认知负荷会挤占学习所需的相关认知负荷 | 节点粒度控制（最小学习单元）、分层设计（不一次性要求学生同时理解概念+掌握技能+熟练操作）、决策索引的上下文极简主义（避免将完整产物注入上下文导致认知/Token 双重过载） |
| **建构主义学习理论（Constructivism, Piaget / Bruner）** | 学习者主动建构知识而非被动接收；同化（纳入已有认知结构）与顺应（调整认知结构以容纳新信息）交替进行；螺旋式课程组织优于一次性完整讲授 | 同层边的前置/后继关系对应同化过程；跨层边的递进关系对应顺应过程；节点内错题闭环触发"认知冲突 → 重构 → 再评估"的微观建构过程 |
| **情境学习理论（Situated Learning, Lave & Wenger）** | 学习本质上是情境性的，知识与技能的习得应嵌入真实使用场景中，而非脱离情境的抽象练习 | 技能层节点设计强调"在不同情境中判断何时使用"；课程结构支持场景式/探究式/项目引领式组织，避免纯章节讲授 |
| **分布式认知理论（Distributed Cognition）** | 认知过程不仅发生在个体头脑中，还分布于工具、环境与他人；智能 Agent 可作为认知的外部资源增强学习 | 多 Agent 协同（规划/备课/虚拟教室/错题闭环）即分布式认知的实现——每个 Agent 承担部分认知任务，与教师/学生共同完成教学流程 |
| **元认知理论（Metacognition, Flavell）** | 学习者对自身认知过程的监控与调节能力是学习成效的关键预测因子 | 节点推荐引擎给出"为什么推荐"的解释以培养学生的路径选择反思；教学评估指标体系引导教师与学生共同审视学习过程 |
| **分层教学理论（Differentiated Instruction, Tomlinson）** | 教师应根据学生的准备程度、兴趣、学习偏好调整内容、过程与产物，而非"一刀切" | 同主题多路径的知识网络设计；学生可在工具层/技能层/概念层之间选择适合自身目标的路径；节点推荐引擎考虑学生的目标层次进行差异化推荐 |

### 1.4 术语约定

| 术语 | 定义 |
|-----|------|
| Agent / 智能体 | 能够自主感知、决策并执行任务的 AI 实体 |
| 多智能体协同 | 多个 Agent 通过消息/协议共同完成目标的过程 |
| 场景（Scene） | 具有明确目标与约束的教学任务类型（如课程规划、虚拟教室） |
| 配置（Config） | 描述 Agent 组合与协作方式的声明式定义 |
| 人机协同（Human-in-the-Loop） | 人类教师在关键节点对 Agent 进行审批、反馈或干预 |
| DeepAgents | LangChain 官方推出的生产级 Agent Harness |
| AgentScope | 阿里通义实验室推出的多智能体一站式开发框架 |
| **知识网络（Knowledge Network）** | 由知识点节点和边关系组成的有向图，描述课程内容的结构化组织方式。不同于线性章节结构，知识网络中从起点到终点存在多条路径 |
| **节点（Node）** | 知识网络中的最小学习单元。一个抽象知识点（如"二元一次方程"）可在三个不同层次上投影为 1~3 个节点 |
| **概念层（Concept Layer）** | 节点的最高层次，关注"为什么"——理解本质、原理、推导与内在机制。对应布鲁姆分类学中的分析/评价/创造层级 |
| **技能层（Skill Layer）** | 节点的中间层次，关注"何时用"——能在不同情境中判断何时使用、灵活迁移、选择策略。对应布鲁姆分类学中的应用/分析层级 |
| **工具层（Tool Layer）** | 节点的基础层次，关注"怎么操作"——掌握固定操作步骤、标准流程、模板化使用。对应布鲁姆分类学中的记忆/理解/应用层级 |
| **立体分层知识网络** | 知识网络的每个抽象知识点在概念/技能/工具三个层次上分别投影为独立节点，形成三维结构。学习路径可在同一知识点内跨越层次推进 |
| **同层边（Intra-layer Edge）** | 连接同一层次中两个节点的边，表示同层次上的前置/后继关系（如"一元一次方程的解法" → "二元一次方程的解法"） |
| **跨层边（Inter-layer Edge）** | 连接不同层次上相关节点的边，表示从工具到技能、或技能到概念的递进关系（如"消元法操作" → "建模策略选择"） |
| **节点掌握（Mastery）** | 学生在某个节点上达到预设评估标准（不同层次的评估标准不同）后的状态标记。掌握后节点推荐引擎被触发 |
| **节点推荐引擎（Node Recommendation Engine）** | 后台决策服务，每次学生在某节点上被标记为"掌握"后，基于知识网络依赖关系 + 学生历史表现 + 学生目标层次，推荐 2~4 个候选下一节点 |
| **节点内错题闭环（In-node Error Loop）** | 学生在节点练习中答错后，系统触发的微观循环：错误诊断 → 针对性补充讲解 → 同类新题（难度微调）→ 再评估，直到掌握或达到重试上限 |
| **动态学习路径（Dynamic Learning Path）** | 学生在知识网络中留下的实际行进轨迹。路径由每次"节点掌握 → 推荐 → 学生选择 → 移动"的迭代过程动态涌现，而非课前预先规划 |
| **教学评估指标（Teaching Evaluation Metrics）** | 在课程规划阶段设计的、用于后续评估教学过程的量表/问卷/观测量表。与知识网络结构关联，不是事后补做 |
| **知识编译（Knowledge Compilation）** | 将 Agent 生成的完整产物（如课程规划）自动提炼为结构化决策索引的过程。不同于 LLM 摘要压缩，知识编译是模板化、可审计、可精确追踪的——每个决策点有唯一 ID |
| **决策点（Decision Point, DP）** | 知识编译过程中提取的、教师已确认的不可变更决策，如"总课时=12"、"教学方法=探究法+项目引领"。每个决策点有唯一 ID（如 DP-01），后续场景以 DP 索引为依据进行一致性检查和信息引用 |
| **决策索引（Decision Index）** | 一个极简列表，包含当前任务中所有已确认决策点的 ID 和关键信息。长度不随阶段数线性增长，用于后续阶段的上下文注入，替代将完整产物拼入上下文 |
| **Knowledge Wiki** | 知识编译产物的结构化存储形式。以 Markdown 页面组织，每个场景/阶段对应一个页面。页面内容受 Harness Schema 约束，由 Git 管理版本，人类可读、LLM 可消费 |
| **思政融合设计** | 备课阶段与教案/学案同步设计的思政元素融合方案。由 Agent 提出"在哪个节点、融什么、如何融"的建议，由教师确认/修改 |
