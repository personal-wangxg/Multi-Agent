# DeepAgents vs AgentScope 框架对比

## 核心对比

| 维度 | DeepAgents | AgentScope |
|-----|------------|------------|
| **开发团队** | LangChain 官方 | 阿里通义实验室 |
| **开源协议** | MIT | Apache 2.0 |
| **核心定位** | 生产级 Agent Harness | 多智能体一站式开发框架 |
| **设计哲学** | 强编排、流程优先 | 弱编排、释放模型能力 |
| **模型要求** | 中等 | 越强越好 |
| **生态依赖** | LangChain + LangGraph | 独立生态，模型中立 |

## 架构对比

### 分层架构

**DeepAgents**（四层）：
- 入口层（Harness）
- 核心能力层（规划、上下文、子Agent、记忆）
- 工具与执行层
- 生产运行时

**AgentScope**（三层）：
- AgentScope Core（智能体构建与编排）
- AgentScope Runtime（运行部署环境）
- AgentScope Studio（可视化监控工具）

## 核心能力对比

| 能力 | DeepAgents | AgentScope |
|-----|-----------|------------|
| **任务规划** | 内置 write_todos，显式待办列表 | 模型自主推理 |
| **子Agent** | 内置，上下文隔离 | Message Hub 动态编排 |
| **文件系统** | 内置 VFS | 需自定义 |
| **持久记忆** | LangGraph Store | ReMe 分层记忆 |
| **沙箱执行** | 内置（Modal/Deno/Daytona） | 需集成 |
| **人机闭环** | 人工审批/编辑/拒绝 | 实时介入控制 |
| **可观测性** | 日志、追踪、审计 | OTel、全链路追踪 |
| **模型微调** | 外部支持 | 内置支持 |

## 部署对比

| 部署方式 | DeepAgents | AgentScope |
|---------|-----------|------------|
| **本地开发** | ✅ | ✅ |
| **Serverless** | 支持 | ✅ |
| **K8s 集群** | 需自行部署 | ✅ 内置支持 |
| **私有化** | 需自行部署 | ✅ |
