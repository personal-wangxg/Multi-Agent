# DP-ARCH-02：9 个典型业务场景

**确认日期**：2026-06-12 · **确认人**：项目核心团队
**决策类别**：架构 · **优先级**：P0

---

## 场景总览

| 阶段 | 场景编号 | 场景名称 | 触发方式 | 本质 |
|-----|---------|---------|---------|-----|
| 课前 | SCENE-001 | **课程规划** | 教师主动发起 | 教师与 Agent 共创：搭建立体分层知识网络 |
| 课前 | SCENE-002 | **备课辅助** | 课程规划完成后，教师按章节发起 | 为知识网络中每个节点生成教案+学案+资源 |
| 课中/课后 | SCENE-003 | **虚拟教室** | 学生主动进入 | 学生在单个节点上的学习体验 |
| 课中/课后 | SCENE-004 | **节点内错题闭环** | 学生在 SCENE-003 中答错时自动触发 | 微观学习过程中的子流程 |
| 课后 | SCENE-005 | **节点推荐引擎** | 学生标记掌握某节点后触发 | 后台决策服务 |
| 课后 | SCENE-006 | **作业批改** | 教师或系统自动发起 | 节点掌握评估 |
| 课后/定期 | SCENE-007 | **学情分析** | 教师或定期自动 | 学生/班级学习画像 |
| 课程末 | SCENE-008 | **教学评估** | 课程/单元结束后教师发起 | 教师教学过程反思 |
| 后台持续 | SCENE-009 | **知识网络动态维护** | 定期自动+教师手动 | 后台 meta-scenario：基于学习数据的网络优化建议 |

## 设计原则

1. **场景之间的数据流动**：前序场景的产物（特别是知识网络）是后续场景的输入基础
2. **SCENE-001 是整个系统的核心入口**：知识网络的质量决定后续所有场景的上限
3. **SCENE-009 是持续进化机制**：基于学生真实数据反哺知识网络优化，形成闭环

## 各场景核心输出物

| 场景 | 核心输出 | 形式 |
|-----|---------|------|
| SCENE-001 | 教学目标体系 + 课程结构 + 立体分层知识网络 + 评估指标 | 结构化 YAML + Wiki 页面 |
| SCENE-002 | 每个节点的教案 + 学案 + 教学资源 + 思政融合方案 | 按节点组织的结构化文档 |
| SCENE-003 | 学生学习过程记录（问题/对话/答题记录） | 会话日志 + 结构化节点状态 |
| SCENE-004 | 错题诊断 + 补充讲解 + 同类新题 + 重新评估 | 错题闭环记录 |
| SCENE-005 | 候选节点推荐列表（2-4个） + 推荐理由 | 节点推荐 JSON |
| SCENE-006 | 批改结果 + 评分 + 点评 + 节点掌握状态 | 批改报告 |
| SCENE-007 | 学生画像 + 班级热力图 + 薄弱环节分析 + 教学建议 | 分析报告 |
| SCENE-008 | 评估报告 + 教学反思 + 改进建议 | 评估文档 |
| SCENE-009 | 知识网络优化建议（节点拆分/合并/新增过渡节点等） | 优化建议报告 + 决策点 |
