# DP-ARCH-05：动态学习路径

**确认日期**：2026-06-12 · **确认人**：项目核心团队
**决策类别**：架构 · **优先级**：P0

---

## 决策摘要

EduAgents 中**不存在课前规划的完整学习路径**。学生的实际学习轨迹是一个**动态涌现过程**：每次学生在某节点上被标记为"掌握"后，节点推荐引擎（SCENE-005）基于知识网络的边关系、学生历史表现、学生目标层次，计算并推荐 2~4 个候选下一节点，**由学生自主选择**后进入虚拟教室（SCENE-003）完成该节点学习。路径由一次次"节点掌握 → 推荐 → 学生选择 → 移动"的迭代过程自然形成。

## 核心设计原则

1. **不做全局路径规划，只推荐下一步**
   - 系统不会在课前生成一条从起点到终点的完整路径
   - 推荐引擎每次仅计算"下一步"的 2~4 个候选节点
   - 路径是动态涌现的，而非预先确定的

2. **知识网络是推荐的主干依据**
   - 候选节点来自当前节点在知识网络中可达的后继节点（沿 is_prerequisite / supports_understanding / enables_operation 边）
   - 网络结构未覆盖的节点不进入推荐范围
   - 跨层边（concept→skill→tool 或反向）支持纵向层次跃迁推荐

3. **学生历史表现加权**
   - 学生在已掌握节点上的正确率、完成速度、错题模式等数据会影响推荐分数
   - 历史数据不足（<3 个节点）时退化为纯网络结构推荐，标注"待积累数据后自动调整"
   - 表现良好的节点类型倾向于推荐下一阶段；表现困难的节点类型倾向于推荐巩固型节点

4. **目标层次匹配**
   - 学生的目标层次（tool / skill / concept）影响候选节点的优先级
   - 以工具层为目标的学生：优先推荐同层的后继节点
   - 以概念层为目标的学生：鼓励沿跨层边向概念层跃迁
   - 推荐理由中明确说明"为什么推荐这个节点"（解释 Agent 负责生成自然语言）

5. **学生拥有最终选择权**
   - 系统推荐的是 2~4 个候选节点，而非一个强制节点
   - 学生可以选择任何一个候选节点，也可以全部拒绝后从知识网络中自由选择
   - 学生的选择会被记录，并在后续推荐中作为个人偏好考虑

6. **推荐可解释，非黑盒**
   - 每个候选节点附带"为什么推荐"的解释文字
   - 解释包括：知识网络依赖（"这是你当前节点的自然后继"）、表现匹配（"你在工具层表现优秀，可以挑战技能层"）、层次匹配（"这符合你以概念理解为目标的学习方向"）等
   - 推荐分数（0~1）也对学生/教师可见

## 关键细节

### 推荐分数的计算逻辑（摘要）

- **基础分（网络结构，60% 权重）**：基于边类型与最短路径长度计算。层内 is_prerequisite 边基础分最高，跨层边视学生目标层次调整
- **表现分（历史数据，30% 权重）**：与当前节点同类节点的历史正确率、掌握速度、错题闭环次数综合
- **层次匹配分（学生目标，10% 权重）**：候选节点 layer 与 student_target_level 的对齐度加分
- **候选筛选**：按推荐分数排序，取前 2~4 个；保证候选节点的层次多样性（不全部集中在同一层次）
- **阈值**：所有候选分数 < 0.5 时扩大搜索范围重新计算；若仍无则提示教师手动建议

### 节点掌握的判定

节点掌握是推荐引擎的触发条件。掌握标准由 stage3 的评估指标设计确定（见 DP-ARCH-03/DP-ARCH-11），典型掌握条件：
- 虚拟教室中节点练习正确率 ≥ 80%
- 错题闭环完成且达到掌握条件（连续 2 次答对同类题 OR 同类题累计正确率 ≥ 80%）
- 作业批改中节点相关题目表现符合评估标准
- 节点完成后由 Harness 校验 mastery_confirmed 字段

### 路径的持久化与回放

- 学生的"实际路径"由一系列（节点 id + 进入时间 + 掌握时间 + 是否掌握）记录组成
- 学情分析（SCENE-007）与教学评估（SCENE-008）可回放学生路径
- 知识网络动态维护（SCENE-009）可基于大量学生的实际路径提出网络优化建议

### 异常与边界

| 情况 | 处理方式 |
|-----|---------|
| 从当前节点出发无可达后继节点 | 提示"你已完成该主题的所有学习！推荐进入下一主题" |
| 所有候选都被学生拒绝 | 列出知识网络中所有其他可选节点，让学生自由选择 |
| 推荐分数全部 < 0.5 | 扩大搜索范围重新计算；若仍无则提示教师手动建议 |
| 学生历史数据不足 < 3 节点 | 纯网络结构推荐，标记"待积累数据后自动调整" |
| 节点未达到掌握但学生想前进 | 不触发推荐引擎，提示先完成当前节点或进入错题闭环 |

## 影响范围

- SCENE-005 节点推荐引擎的全部行为逻辑
- SCENE-003 虚拟教室的节点状态流转（mastery_confirmed 触发推荐）
- SCENE-004 节点内错题闭环的完成条件（影响掌握判定）
- SCENE-007 学情分析中的路径回放与可视化
- SCENE-009 知识网络动态维护中的路径数据分析
- DP-ARCH-04 三层节点结构中的 cross-layer 边对推荐的影响
- DP-ARCH-02 9 个场景的数据流：从节点掌握到推荐的闭环
