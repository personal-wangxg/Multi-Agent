# DP-ARCH-06：功能需求（FR）

**确认日期**：2026-06-12 · **确认人**：项目核心团队 · **决策类别**：架构

---

## 决策摘要

本决策点汇总并确认系统全部 18 项功能需求（FR-01 ~ FR-18）的功能定义及其优先级。这些 FR 覆盖平台基座（FR-01~04）、核心教学能力（FR-05~11）、运营支撑能力（FR-12~14）、约束与合规（FR-15~16）、扩展能力（FR-17）与知识编译核心（FR-18））。所有需求一经确认即成为后续架构与内容决策点的基线，后续设计。

## 核心设计原则

1. **分层优先级管理：P0（必须实现，非实现不可上线）、P1（必须实现）、P2（可选），
2. **分层架构：平台基座层 → 核心教学能力层 → 约束与支撑能力层，
3. **场景驱动，而非功能堆砌；9 类场景作为功能需求的组织依据，
4. **迭代共创原则：教师与 Agent 共同产出不可变更产物，
5. **决策前置编译原则：完整产物经编译为 DP 索引后消费。
