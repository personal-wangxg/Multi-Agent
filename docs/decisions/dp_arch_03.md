# DP-ARCH-03：课程规划三阶段设计

**确认日期**：2026-06-12 · **确认人**：项目核心团队
**决策类别**：架构 · **优先级**：P0

---

## 决策摘要

SCENE-001 课程规划是系统的核心入口场景。该场景采用**三阶段迭代共创**设计：教师输入课程基本信息后，依次经过（1）教学目标体系设计 →（2）课程结构设计 →（3）立体分层知识网络设计三个阶段。每个阶段均遵循"Agent 生成方案 → 教师审阅/修改 → Agent 迭代 → 教师确认"的共创闭环。评估指标体系贯穿三个阶段，与各阶段产物同步产出。

## 核心设计原则

1. **教师与 Agent 共创而非单向输出**
   - 课程方向由人类教师最终确定，Agent 仅作为辅助生成与迭代工具
   - 每个阶段可多轮迭代（方案 v1 → v2 → ... → 教师确认）
   - 不强制一次生成完整方案，允许渐进式设计

2. **阶段间严格依赖而非并行**
   - stage1 教学目标是 stage2 课程结构的输入约束
   - stage2 课程结构（章节/单元）是 stage3 知识网络节点拆分的依据
   - 前序阶段未确认，后序阶段不启动

3. **评估指标贯穿始终而非事后补做**
   - 评估指标设计 Agent 在三个阶段均同步工作
   - stage1 产出教学目标匹配的评估量表
   - stage2 产出与结构单元匹配的观测点/量表
   - stage3 产出与节点绑定的评估项/量表
   - 节点掌握标准（如正确率 ≥ 80%）在规划阶段明确

4. **立体分层而非线性章节**
   - stage3 的知识网络每个抽象知识点在概念/技能/工具三个层次上独立投影为节点
   - 网络支持多起点多路径，而非单一线性章节

5. **结构化产物可审计而非自由文本**
   - 每个阶段产物均输出为结构化 YAML 格式
   - 包含必要的元数据：course_name、target_level、total_periods、version、teacher_id
   - 由 Harness Schema 校验，字段缺失时自动重试

## 关键细节

### 阶段输入输出摘要

| 阶段 | 主要 Agent | 输入 | 核心输出 |
|-----|-----------|-----|---------|
| Stage 1 | 教学目标设计 Agent + 评估指标设计 Agent | 教师输入的课程基本信息（名称/描述/目标层次/课时） | 知识目标 + 能力目标 + 素质目标 + 预期学习效果 + 教学方法选择（8种中选择） + 学习效果检验方式 + 匹配评估量表 |
| Stage 2 | 课程结构设计 Agent + 评估指标设计 Agent | stage1 产物（特别是教学方法选择） | 章节/场景/项目式结构单元列表 + 每单元课时 + 核心知识点关键词 + 建议教学法 + 与结构单元匹配的观测点/量表 |
| Stage 3 | 知识网络构建 Agent + 评估指标设计 Agent | stage2 产物（结构单元与关键词映射） | 概念层节点 + 技能层节点 + 工具层节点 + 层内边 (is_prerequisite) + 跨层边 (supports_understanding/enables_operation) + 同主题边 (same_topic_cross_layer) + 与节点绑定的评估项/量表 |

### 节点基础字段

每个节点（概念/技能/工具层）必须包含：
- `id`：节点唯一标识（命名建议：`{layer}_{topic}_{detail}`，如 `tool_formula_method`）
- `layer`：concept / skill / tool（三选一）
- `title`：节点名称（教师可读）
- `bloom_level`：remember / understand / apply / analyze / evaluate / create（布鲁姆分类层级）
- `difficulty`：1~5（难度等级）
- `can_self_learn`：true / false（是否支持自学掌握）
- `estimated_periods`：预计课时
- `mapped_unit`：关联 stage2 中的结构单元 id
- `teaching_objectives`：关联该节点的知识/能力/素质目标

### 迭代与异常处理

- 单阶段迭代超过 5 次仍未确认：记录为"阶段性搁置"，提示教师可能需重新审视本阶段设计范围
- 教师连续 3 次对同一方案提出修改：记录为"争议方案"，标记可能需要教师进一步澄清设计意图
- Token 超预算：将阶段产物拆分输出后重试（如先输出节点再输出边）
- 阶段产物通过 Harness Schema 校验后才能持久化
