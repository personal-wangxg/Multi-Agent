# 选型建议

## 决策树

```
是否需要严格流程控制？
├── 是 → 选择 DeepAgents（显式规划、待办列表）
└── 否 → 是否需要模型自主推理？
    ├── 是 → 选择 AgentScope（释放优先）
    └── 否 → 根据其他因素决定
```

## 选择 DeepAgents 的场景

### 1. 需要显式任务规划
- 长周期、多步骤任务
- 需要清晰的任务分解和状态追踪
- 任务流程需要可追溯、可审计

### 2. 需要内置文件系统
- 处理大上下文任务
- 需要持久化中间结果
- 突破 token 窗口限制

### 3. 需要沙箱执行
- 执行 shell 命令、代码
- 需要安全隔离环境
- 自动化测试、构建、部署

### 4. 已有 LangChain 生态
- 已有 LangChain/LangGraph 代码
- 需要平滑迁移
- 团队熟悉 LangChain 生态

### 5. 需要快速上手
- 一行代码创建完整 Agent
- 开箱即用的生产级能力
- 低代码开发

## 选择 AgentScope 的场景

### 1. 需要灵活多Agent编排
- 动态任务交接
- 多角色智能体自主分工
- 复杂工作流编排（Message Hub）

### 2. 需要本地/私有化部署
- K8s 集群部署
- 私有化部署需求
- 企业级安全合规要求

### 3. 需要模型微调
- 特定领域优化 Agent 行为
- 内置微调支持
- 模型能力强，想充分发挥

### 4. 需要全生命周期管理
- 开发、部署、监控一体化
- 可视化开发工具（Studio）
- 全链路可观测性

### 5. 需要实时介入控制
- 紧急任务插入
- 动态流程调整
- 人工审批关键操作

## 两者同时使用的场景

### 混合架构方案

```
                    用户请求
                        │
                        ▼
              ┌───────────────────┐
              │   AgentScope      │
              │   (消息中枢)       │
              └─────────┬─────────┘
                        │
        ┌───────────────┼───────────────┐
        ▼               ▼               ▼
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│ DeepAgents  │  │ DeepAgents  │  │   其他Agent │
│  (研究型)   │  │  (开发型)   │  │    类型     │
└─────────────┘  └─────────────┘  └─────────────┘
```

### 适用场景

- 需要多种智能体类型协作
- 部分任务需要严格流程控制，部分需要灵活自主
- 利用 AgentScope 的消息中枢统一管理
- 利用 DeepAgents 的专业能力处理特定任务

### 优势

- 发挥各自优势
- 灵活组合
- 渐进式架构演进

## 总结建议

| 需求优先级 | 推荐框架 |
|-----------|---------|
| 显式规划 + 文件系统 | DeepAgents |
| 灵活编排 + 私有化部署 | AgentScope |
| LangChain 生态深度集成 | DeepAgents |
| 多Agent协作 + 实时干预 | AgentScope |
| 快速原型 + 低代码 | DeepAgents |
| 企业级全生命周期 | AgentScope |
